# CLF-002(AWS Certified Cloud Practitioner) 덤프 풀기

## 시험 내용

AWS 클라우드의 가치 설명

- AWS 공동 책임 모델 이해 및 설명
- 보안 모범 사례 이해
- AWS 클라우드 비용, 경제성 및 결제 관행 이해
- 컴퓨팅, 네트워크, 데이터베이스, 스토리지 서비스를 포함한 핵심 AWS 서비스 설명 및 포지셔닝
- 일반 사용 사례에 맞는 AWS 서비스 파악

## 답안 유형

- 선다형: 정답 1 개와 오답 3 개(정답 이외의 답)가 있습니다.

## 자주 나오는 개념

### AWS Well-Architected Framework(프레임워크)

- 운영 우수성 : `운영 및 모니터링` 프로세스르 `점검하며 개선`을 통해 지속적인 통합, 제공을 지향 함
  - 코드 기반의 운영
  - 소규모 변경 사항에 대한 반복 적용
  - 실패 예측 및 학습
- 보안 : 데이터, 시스템 및 자산을 보호하는 방안`을 `안내`하며 보안 설정 및 서비스에 대한 적용 현황 점검
  - 강력한 자격 증명 기반 구현
  - 추적 기능 활성화
  - 모든 계층에 보안 적용
  - 전송 및 저장 데이터의 보호
  - 데이터의 엑세스 제어
- 안전성 : 워크로드의 `일관성, 장애 복구 프로세스 현황 점검`
  - 장애에 대한 자동 복수
  - 복구 절차의 테스트
  - 수평적 확정으로 단일 장애에 대해 대비(가용성 증가)
- 성능 효율성 : 클라우드 `리소스를 요구사항에 맞게 효율적으로 사용하고 유지` 할수 있는지 점검
  - 서버리스 아키텍처 사용
  - 자동화를 통한 반복 테스트 수정
- 비용 최적화 : `불필요한 비용 발생 방지`를 중점에 두고 점검
  - 클라우드 재무 관리 구현
  - 소비 모델 도입
- 지속 가능성 : `환경 영향, 에너지 소비 효율성`에 중점을 두고 점검
  - 관리형 서비스 사용
  - 지속 가능성 목표 수립
- ⭐️ AWS Well-Architected Framework의 권장되는 설계 원칙은 무엇입니까?
  - 운영 실패로부터 개선하는 법을 배우세요( Learn to improve from operational failures. )
- ⭐️ AWS Well-Architected Framework의 운영 우수성 기둥에 포함되는 설계 원칙은 무엇입니까?
    - 실패를 예상하세요.


### AWS WAF

- 용성에 영향을 미치거나 보안을 훼손하거나 과도한 리소스를 소비할 수 있는 일반적인 웹 익스플로잇 및 봇으로부터 애플리케이션을 보호하도록 도와줍니다.
- ✅ `Well-Architected Framework`와 다르다!!!!
- ⭐️ SQL 주입 공격을 차단에 사용되는 AWS 기능

### AWS Cloud Adoption Framework(AWS CAF)

- 조직이 클라우드 기술을 시작하는 데 도움이 되는 일련의 모범 사례, 도구 및 지침입니다.
- 조직이 워크로드를 클라우드로 이전할 때 위험을 식별 및 완화하고 비용을 관리하며 규정 준수를 보장하는 데 도움을 줌
- 효율성 향상을 위해 클라우드에서 거버넌스 및 보안을 최적화하는 방법에 대한 지침도 제공.
- ⭐️ 기본 기능을 포함하는 관점은 무엇입니까?

### AWS Service Catalog

- AWS에서 사용이 승인된 IT 서비스 카탈로그를 생성하고 관리할 수 있습니다.
  - IT 서비스에는 가상 머신 이미지, 서버, 소프트웨어 및 데이터베이스에서 멀티 티어 애플리케이션 아키텍처를 완성하는 모든 서비스가 포함될 수 있습니다.
- 직은 일반적으로 배포되는 IT 서비스를 중앙에서 관리하고, 조직이 일관된 거버넌스를 달성하고 규정 준수 요구 사항을 충족할 수 있습니다. 최종 사용자는 조직에서 규정한 제약에 따라, 필요에 따라 승인된 IT 서비스만 신속하게 배포할 수 있습니다.
- ⭐️ IT 서비스를 관리하고 코드(IaC) 템플릿으로 인프라를 관리 -> 어떤 AWS 서비스를 사용해야 할까?

### Amazon Athena

- 표준 SQL을 사용하여 Amazon S3(Amazon Simple Storage Service)에 있는 데이터를 직접 간편하게 분석할 수 있는 대화형 쿼리 서비스입니다.

### AWS Security Token Service(AWS STS)

- AWS 리소스에 대한 액세스를 제어할 수 있는 임시 보안 자격 증명을 생성하여 신뢰받는 사용자에게 제공할 수 있습니다.
- 임시 보안 자격 증명은 그 이름이 암시하듯 단기적입니다.
- ⭐️ 애플리케이션은 다른 AWS API를 인증하기 위해 임시적이고 제한된 권한의 자격
 증명을 사용해야 합니다. 이러한 인증 요구 사항을 충족하기 위해 어떤 AWS 서비스를 사용하는가?

### AWS Secrets Manager

- API 키, 비밀번호, 데이터베이스 자격 증명과 같은 민감한 정보를 관리하도록 특별히 설계되었습니다.
- 보안을 강화하고 운영 오버헤드를 줄이기 위해 비밀을 자동으로 순환하는 기능이 포함되어 있습니다.
- ⭐️ 데이터베이스 사용자 자격 증명을 순환하기 위한 일정을 수립하려고 합니다. 어떤 AWS 서비스가 최소한의 운영 오버헤드로 이 요구 사항을 지원하는가?

### AWS Key Management Service(AWS KMS)

- 데이터 암호화에 사용되는 암호화 키를 쉽게 생성, 제어 및 관리할 수 있는 완전 관리형 서비스입니다.
- 암호화 목적으로 Amazon EBS를 포함한 다른 AWS 서비스와 완벽하게 통합됩니다.
- ⭐️ Amazon EBS에 대한 암호화를 제공하기 위해 어떤 AWS 서비스를 사용합니까?

### AWS CloudFormation

- 안전하고 예측 가능하며 반복 가능한 방식으로 AWS 인프라를 코드로 정의하고 프로비저닝할 수 있는 서비스입니다.
- AWS 인프라를 코드로 정의하고 프로비저닝할 수 있는 서비스입니다..
- 개발 및 프로덕션 환경을 일관되게 유지하는 데 도움이 됩니다.
- ⭐️ 개발 환경 인프라와 프로덕션 환경 인프라를 반복 가능한 방식으로 유지 관리하기 위한 서비스

### AWS Artifact

- AWS 규정 준수 보고서, 인증 및 증명에 대한 주문형 액세스를 제공하는 포털입니다.
- 다양한 규정 준수 관련 문서에 대한 중앙 집중화된 서비스이다.
- ⭐️ 사용자가 AWS 규정 준수 관련 정보 및 보고서를 찾을 수 있는 가장 좋은 리소스는 무엇입니까?
  - AWS Artifact

### AWS Global Accelerator

- AWS 글로벌 네트워크를 사용하여 애플리케이션으로의 트래픽 라우팅을 최적화하는 서비스입니다.
- 애니캐스트 IP 주소를 활용하여 애플리케이션의 가용성과 성능을 개선합니다. 특히 네트워크 성능을 전역적으로 개선합니다.
- ⭐️ 전 세계 네트워크 인프라를 통해 트래픽을 전송하여 네트워크 성능을 개선합니까?

### AWS Security Hub

- 다양한 AWS 서비스와 통합 파트너 제품에서 보안 결과를 집계하고 우선순위를 지정하는 클라우드 보안 포스처 관리(CSPM) 서비스입니다.
- AWS 리소스에 대한 자동화된 지속적인 보안 모범 사례 검사를 수행하여 잘못된 구성을 식별하고 보안 경고(즉, 결과)를 표준화된 형식으로 집계하여 보다 쉽게 ​​보강, 조사 및 수정할 수 있도록 돕는 클라우드 보안 포스처 관리(CSPM) 서비스입니다.

### Amazon Macie

- S3에서 개인 식별 정보(PII)와 같은 민감한 데이터를 자동으로 검색, 분류 및 보호하기 위해 머신 러닝을 사용하는 보안 서비스입니다
- ⭐️ 어떤 AWS 서비스가 사용자에게 Amazon S3 버킷에 저장된 중요한 데이터를 검색하고 보호할 수 있는 기능을 제공합니까?

### AWS CloudTrail

- AWS 계정의 API 호출 및 관련 이벤트를 기록하는 서비스입니다.
- 사용자 활동, 리소스 변경 및 중요 이벤트에 대한 가시성을 제공합니다
- AWS 계정의 운영 및 위험 `감사`, 거버넌스 및 규정 준수를 활성화하는 데 도움이 되는 AWS 서비스입니다.
- ⭐️ 어떤 AWS 서비스가 Amazon EC2 인스턴스가 종료된 시점을 식별할 수 있습니까?

### Amazon RDS

- PostgreSQL을 포함한 다양한 데이터베이스 엔진을 지원하고 OLTP 워크로드에 적합한 관리형 데이터베이스 서비스를 제공합니다.
- 온라인 트랜잭션 처리(OLTP) 워크로드에 적합합니다.
- ⭐️ 온라인 트랜잭션 처리(OLTP)를 제공하는 관리형 PostgreSQL 데이터베이스를 실행할 수 있는 AWS 서비스는 무엇입니까?

### AWS CloudShell

- AWS에서 제공하는 웹 브라우저 기반 `셸`로, AWS Management Console에서 직접 AWS 리소스에 명령줄로 액세스할 수 있습니다.
- 사용자는 AWS CloudShell을 사용하여 추가 소프트웨어를 설치하지 않고도 AWS CLI 명령을 실행하고 다양한 AWS 도구를 사용할 수 있습니다.
- ⭐️ 웹 브라우저에서 직접 AWS 도구 및 리소스에 대한 명령줄 액세스를 제공하는 AWS 서비스는 무엇입니까?

### AWS Management Console(AWS 관리 콘솔)

- 사용자가 그래픽 사용자 인터페이스(GUI)를 사용하여 AWS 리소스에 액세스하고 관리할 수 있는 웹 기반 인터페이스입니다.
- ⭐️ 웹 `인터페이스`를 통해 AWS 클라우드 리소스를 관리하려고 합니다. 어떤 AWS 서비스가 이 요구 사항을 충족할까요?

### AWS 가격 계산기

- 사용자가 AWS 서비스 사용 비용을 추정할 수 있는 웹 기반 도구입니다
- 사용 패턴, 지역 및 기타 매개변수를 기반으로 다양한 AWS 리소스와 관련된 비용을 이해하고 추정하는 데 도움이 됩니다.
- `AWS 비용 탐색기` <- 현재, 과거 사용했던 비용 및 사용량에 대한 시각화 및 분석 제공


### Amazon Connect
- 고객을 대신하여 전 세계의 전화 통신 공급자 네트워크를 관리합니다.
- 😒 EC2 인스턴스에 연결하기 위해 사용하는 서비스가 아니다!!

  이 회사는 Amazon RDS 배포 영역을 선택하기 위해 어떤 AWS 서비스 또는 리소스를 사용해야 합니까?

### AWS Snowball
- 대규모 데이터 전송을 위해 특별히 설계된 서비스입니다.
- 데이터를 Snowball 어플라이언스에 로드하여 AWS로 다시 배송하면 AWS에서 해당 데이터를 S3 버킷으로 전송합니다.

### Concierge Support team

### AWS X-Ray
- 클라우드 애플리케이션을 지원하는 AWS 리소스의 `추적 데이터`를 사용하여 상세한 `추적 맵`을 생성합니다.
- 개발자가 마이크로서비스 아키텍처를 사용하여 구축된 것과 같은 프로덕션, `분산 애플리케이션`을 분석하고 디버깅하는 데 도움이 됩니다. 

### AWS Inspector
- 워크로드를 자동으로 발견하고 소프트웨어 `취약성`과 의도하지 않은 네트워크 노출 여부를 지속적으로 스캔하는 취약성 관리 서비스입니다.


### Cloud Watch

### Amazon GuardDuty

### AWS Organizations
- 생성한 여러 AWS 계정을 조직에 통합하고 중앙에서 관리할 수 있는 계정 관리 서비스입니다. 
- 계정 관리 및 통합 결제 기능을 활용하면 기업의 예산, 보안 및 규정 준수 요구 사항을 보다 잘 충족할 수 있습니다
- 조직의 관리자로서 조직에서 계정을 생성하고 기존 계정을 조직에 초대할 수 있습니다.

### Amazon Lightsail

- 웹 어플리케이션, DB 등 프로젝트를 빠르게 시작하는 데 필요한 모든 것을 저렴하고 예측 가능한 월별 요금으로 제공합니다.

### AWS Storage Gateway
- 온프레미스 애플리케이션이 클라우드 스토리지를 원활하게 사용할 수 있도록 하는 하이브리드 클라우드 스토리지 서비스입니다.
- 온프레미스 환경을 AWS 클라우드 스토리지와 통합하기 위한 파일, 볼륨 및 테이프 게이트웨이 인터페이스를 제공합니다.
- 온프레미스 사용자가 하이브리드 스토리지 인프라를 유지하면서 사실상 무제한의 클라우드 스토리지에 액세스할 수 있습니다.
